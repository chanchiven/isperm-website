# ✅ GitHub Pages 部署前检查清单

在部署到 GitHub Pages 之前，请完成以下检查：

## 📋 必须完成的步骤

### 1. 配置文件检查
- [x] `next.config.js` 已配置 `output: 'export'`
- [x] `next.config.js` 已配置 `images.unoptimized: true`
- [ ] 如果仓库名不是 `username.github.io`，已配置 `basePath`
- [x] `.github/workflows/deploy.yml` 已创建

### 2. GitHub 仓库设置
- [ ] 已在 GitHub 创建仓库
- [ ] 仓库设置为 **Public**（免费 GitHub Pages 需要）
- [ ] 已推送代码到 GitHub
- [ ] 已在 Settings > Pages 中启用 GitHub Actions

### 3. 本地测试
- [ ] 运行 `npm run build` 成功（无错误）
- [ ] 检查 `out/` 目录已生成
- [ ] 测试本地构建的静态文件（可选）

### 4. 功能限制确认
- [ ] 了解 API 路由（联系表单）在静态部署中无法工作
- [ ] 已决定如何处理联系表单（使用第三方服务或更换托管平台）
- [ ] 了解中间件功能限制

### 5. 域名配置（如果使用自定义域名）
- [ ] DNS 记录已配置（CNAME 指向 `username.github.io`）
- [ ] `CNAME` 文件中的域名正确
- [ ] 已在 GitHub Pages 设置中添加自定义域名

## 🔍 部署后验证

部署完成后，检查：

- [ ] 网站可以正常访问
- [ ] 所有语言路由正常工作（/en, /es, /ar 等）
- [ ] 图片正常显示
- [ ] CSS 样式正常加载
- [ ] 导航链接正常工作
- [ ] 语言切换器正常工作
- [ ] 移动端显示正常（响应式布局）

## ⚠️ 已知限制

- ❌ 联系表单 API 无法工作（需要替代方案）
- ⚠️ 服务器端中间件功能受限（语言检测、中文用户 404）
- ✅ 多语言静态页面正常生成和访问

## 🆘 如果遇到问题

1. 检查 GitHub Actions 日志
2. 查看浏览器控制台错误
3. 确认 `basePath` 配置是否正确
4. 参考 `GITHUB_PAGES_DEPLOYMENT.md` 详细文档

---

**完成所有检查后，即可开始部署！** 🚀
